# Frontend Tasks

SyncNesto Frontend の実装状況と次タスクを整理するためのメモです。

## 実装済み

### コンポーネント配置

feature 配下の `components/` は、ページ単位と用途単位で分類します。

```text
features/{feature}/components/
├─ pages/      app/ から直接呼ばれるページ単位コンポーネント
├─ forms/      入力フォーム
├─ tables/     一覧テーブル
└─ shared/     その feature 内で複数画面から使う部品
```

`features/projects` は画面の責務が分かれるため、以下の分類を使います。

```text
features/projects/components/
├─ management/ プロジェクト管理画面
├─ joined/     参加プロジェクト画面
├─ forms/      入力フォーム
├─ tables/     一覧テーブル
└─ shared/     プロジェクト feature 内の共通部品
```

### 1. 競合解決モーダル

409 `VERSION_CONFLICT` が返ったとき、競合した項目を一覧表示し、項目ごとに「自分の入力」または「サーバー最新値」を選択して再送信できるようにします。

対象:

- アカウント更新
- ユーザー更新
- プロジェクト更新
- プロジェクトメンバー権限更新
